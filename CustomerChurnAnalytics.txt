Customer Churn Analytics + Business Dashboard

1) Problem Statement
Telecom companies lose revenue when customers cancel their subscription. This system predicts churn risk and converts predictions into actionable business recommendations.

2) Production Architecture (Minimal and Stable)
User (Streamlit) -> POST /predict -> FastAPI -> Feature Engineering -> Model -> Response

Key rules:
- No CSV loading at runtime
- No training at runtime
- No SHAP or heavy compute in production
- Model is loaded once from models/best_model.joblib

3) Tech Stack
- Python 3.11
- FastAPI + Uvicorn
- Streamlit
- Pandas, NumPy, scikit-learn, Joblib

4) Runtime Files (Keep Only)
- src/api.py
- src/dashboard.py
- src/features.py
- src/insights.py
- src/schemas.py
- src/config.py
- models/best_model.joblib
- requirements.txt

5) API Behavior
Endpoints:
- GET / -> {"message": "API is running"}
- GET /health -> {"status": "ok" | "model_not_loaded"}
- POST /predict -> returns churn probability, label, and insights

Input:
- Send raw customer attributes
- API applies feature engineering before inference
- Missing fields are filled safely

6) Dashboard Behavior
- Collects customer inputs via form
- Sends request to deployed API
- Displays churn probability and insights

7) Local Run
1. Create venv and install dependencies
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
2. Run API
   uvicorn src.api:app --host 0.0.0.0 --port 8000
3. Run dashboard
   streamlit run src/dashboard.py

8) Render Deployment
API Start Command:
   uvicorn src.api:app --host 0.0.0.0 --port $PORT

Dashboard Start Command:
   streamlit run src/dashboard.py --server.port $PORT --server.address 0.0.0.0
